Dedicated `/habit/[id]` route with 30-day completion heatmap, AI plan display, and full refine/regenerate interaction — moving plan depth off the dashboard into a focused single-habit view.

## What Was Built

**page.tsx** — Minimal Next.js App Router shell: "use client", extracts `params.id`, renders `<HabitDetailClient habitId={params.id} />`.

**HabitDetailClient.tsx** — Full habit detail page component:
- Auth guard redirects to "/" if no session
- Parallel data fetches: habit row, active plan (via `/api/ai/plan/list`), last-30-days habit_logs (heatmap), total completion count, today's log status
- Back arrow row navigating to /dashboard
- Habit header card (glass className): icon + serif name + accepted_amount subtitle + today's status badge (green "✓ Completed today" or muted "Not logged today")
- Completion count line ("Completed N times")
- 30-day dot heatmap: 10×10 amber circles for completed days, faded surface-border dots for missed days
- Plan section (when plan exists): week focus amber card, core philosophy italic, numbered action steps, 2-col 4-week roadmap grid
- Refine flow: toggle textarea → submit → streaming preview → Approve saves + refreshes / Discard clears
- Regenerate flow: "Refresh plan" → confirm dialog → generate-stream → save → refresh plan state
- No plan state: muted centered message
- Loading state: centered "Loading..." with flex centering
