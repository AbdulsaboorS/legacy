# Phase 5.1: Habit Detail Page - Context

**Gathered:** 2026-03-18
**Status:** Ready for planning

<domain>
## Phase Boundary

Create a dedicated `/habit/[id]` page that houses the full AI plan content currently expanded inline on the dashboard. Dashboard habit cards keep the check-in toggle in place — only the plan view moves to the detail page. Dashboard becomes a pure check-in list.

</domain>

<decisions>
## Implementation Decisions

### Dashboard card behavior
- Check-in (checkmark circle) stays on the dashboard card — tapping it logs the habit as before
- The `✨ Your Plan` row (currently the expand/collapse toggle) becomes a navigation link to `/habit/[id]`
- Habits with a plan show the `Your Plan` row → navigates to detail page
- Habits without a plan show `✨ Want a personalized plan?` button → generates inline on dashboard (current flow unchanged)
- After plan saves, the `Your Plan` nav row appears on the dashboard card
- The old inline expand/collapse section is fully removed — no more in-place plan expansion on dashboard

### Detail page scope
- **Plan content** — This week's focus, core philosophy, action steps, 4-week roadmap (moved from dashboard inline)
- **Log history** — 30-day calendar heatmap of completions for this specific habit (day dots colored by completion, like a minimal GitHub contribution grid); fetches `habit_logs` filtered to this `habit_id`
- **Per-habit completion count** — e.g. "Completed 18 times" from a COUNT on `habit_logs` for this habit
- No check-in button on the detail page — check-in stays on dashboard only
- Detail page only accessible for habits that have a plan (no plan = no `Your Plan` nav row = no navigation)

### Plan interactions (on detail page only)
- Refine, Regenerate, and all related UI (approve, discard, confirm dialog) move to `/habit/[id]`
- All plan interaction state (`refineHabitId`, `refinedPlan`, `showRegenerateConfirm`, etc.) is removed from DashboardClient
- Generation still streams inline on the dashboard (for habits with no plan yet) — unchanged

### Page header
- Back arrow (`← Back` or `‹`) in top-left, links back to `/dashboard`
- Browser back also works
- Full habit card summary at top: habit icon, habit name (Cormorant Garamond serif heading), `accepted_amount` or `suggested_amount` subtitle, and today's completion status (read-only indicator, not interactive)

### Navigation & entry points
- Entry: `Your Plan` row on dashboard habit card → `router.push('/habit/[habitId]')`
- Return: explicit back arrow in header + browser back
- No other entry points in this phase

### Claude's Discretion
- Exact heatmap grid layout (7-day rows vs single row of 30 dots)
- Loading state for the detail page
- Exact sizing and spacing of the habit summary header section
- How the back arrow is styled (icon choice, position)

</decisions>

<specifics>
## Specific Ideas

- The detail page is a focused, calm view of one habit's plan — not a busy dashboard
- The `Your Plan` row on the dashboard card should feel like a navigation affordance, not an expand toggle (chevron `›` pointing right, not `▾` pointing down)

</specifics>
