```css
/* ========================================
   iOS Safe Area Insets (Capacitor)
   Prevents content from hiding behind
   notch, Dynamic Island, and home bar.
   Only takes effect when viewport-fit=cover
   is set. On web these return 0.
   ======================================== */
body {
  padding-top: env(safe-area-inset-top);
  padding-bottom: env(safe-area-inset-bottom);
  padding-left: env(safe-area-inset-left);
  padding-right: env(safe-area-inset-right);
}
```

Important: The beta banner in layout.tsx is `position: fixed, top: 0`. On iOS, after adding body padding-top for safe area, the banner's `top: 0` will correctly align to the safe area edge because the Capacitor WebView respects the safe area coordinate system when `viewport-fit=cover` and `overlaysWebView: false` are both set. No additional changes to the banner needed for the initial build — verify on device.</action>
