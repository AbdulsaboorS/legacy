<task type="checkpoint:human-action">
  <name>Step B: Generate iOS project and configure Info.plist</name>
  <what-built>capacitor.config.ts is ready. This step generates the ios/ directory and configures the URL scheme for deep links.</what-built>
  <how-to-verify>
    Run these commands from the project root:

    ```bash
    # Generate the iOS Xcode project
    npx cap add ios

    # Sync web assets and plugin config to the iOS project
    npx cap sync ios
    ```

    Then edit `ios/App/App/Info.plist` — add the CFBundleURLTypes entry for the deep link scheme. Find the `<dict>` section and add before the closing `</dict>`:

    ```xml
    <key>CFBundleURLTypes</key>
    <array>
      <dict>
        <key>CFBundleTypeRole</key>
        <string>Editor</string>
        <key>CFBundleURLSchemes</key>
        <array>
          <string>app.joinlegacy</string>
        </array>
      </dict>
    </array>
    ```

    Verify: `grep -c "app.joinlegacy" ios/App/App/Info.plist` returns 1.

    Type "done" when Info.plist is saved.
  </how-to-verify>
  <resume-signal>Type "done" to proceed to Xcode device run</resume-signal>
</task>

<task type="checkpoint:human-action">
  <name>Step C: Run on real iPhone device via Xcode</name>
  <what-built>ios/ project is generated and Info.plist is configured.</what-built>
  <how-to-verify>
    ```bash
    npx cap open ios
    ```

    In Xcode:
    1. Connect iPhone via USB
    2. Select your iPhone as the build target (not a simulator)
    3. Set signing: Signing & Capabilities > Team = your Apple Developer account
    4. Build and run (Cmd+R)
    5. Trust the developer certificate on iPhone (Settings > General > VPN & Device Management)

    Verify on device — check each item:

    **App launch:**
    - App launches and loads the Legacy landing page
    - console.log shows: [NativeAuthHandler] platform: ios isNative: true

    **OAuth deep link (MOB-02):**
    - Tap "Continue with Google" — Safari opens Google sign-in
    - After Google auth completes, app opens
    - New user (no habits): lands on /onboarding
    - Returning user (has habits): lands on /dashboard

    **Safe area (MOB-03):**
    - No content hidden behind the notch or Dynamic Island at the top
    - Home bar area at the bottom is not covered by navigation or content
    - Open the refine screen, tap a text input — keyboard appears without pushing content off-screen

    **Session persistence — backgrounding (MOB-04):**
    - Background app for 1+ minute (press home button, use another app)
    - Tap the Legacy icon to restore the app
    - Confirm you land on /dashboard still authenticated (not redirected to the landing page)

    **Session persistence — full app kill (MOB-04 edge case):**
    - Swipe the app fully off the screen in the iOS app switcher (complete kill)
    - Relaunch Legacy from the home screen
    - Confirm /dashboard loads authenticated without requiring sign-in again

    NOTE — if the full-kill test fails (redirected to landing page instead of dashboard):
    This means the Capacitor WebView cookie store does not persist across app kills, which
    would affect users who fully close the app. Escalate to Claude with this message:
    "Session lost on full app kill — switch the Supabase client factory to return the
    native.ts localStorage client when Capacitor.isNativePlatform() is true, so session
    tokens survive WebView cookie resets."
    Do NOT proceed to Step D until this is resolved or explicitly accepted as a known limitation.

    Type "verified" if all checks pass, or describe any issues.
  </how-to-verify>
  <resume-signal>Type "verified" or describe issues</resume-signal>
</task>

<task type="checkpoint:human-action">
  <name>Step D: Codemagic setup and TestFlight build</name>
  <what-built>codemagic.yaml exists and is committed to the repo.</what-built>
  <how-to-verify>
    1. **Codemagic account**: Sign up at codemagic.io (free tier: 500 Mac M1 min/month)

    2. **Connect repository**: Add your GitHub repo to Codemagic

    3. **Upload signing assets** (Team > Code Signing):
       - Distribution certificate (.p12 + password) — export from Keychain Access or Apple Developer portal
       - Provisioning profile — download from Apple Developer portal after creating one for app.joinlegacy

    4. **App Store Connect API key** (Team > Integrations > App Store Connect):
       - Create API key at appstoreconnect.apple.com > Users and Access > Integrations > App Store Connect API
       - Upload issuer ID, key ID, and .p8 file to Codemagic under the name "codemagic"

    5. **Environment variable**:
       - Team > Global variables: APP_STORE_APPLE_ID = [numeric ID from Step A]

    6. **Trigger a build**: Start build from Codemagic dashboard

    7. **TestFlight**: Once build uploads, go to appstoreconnect.apple.com > TestFlight, add yourself as internal tester, install via TestFlight app

    Verify:
    - Codemagic build completes green
    - IPA uploaded to TestFlight
    - App installs from TestFlight on iPhone
    - All Step C verifications pass on the TestFlight build

    Type "shipped" when TestFlight build is verified.
  </how-to-verify>

<task type="checkpoint:human-action">
  <name>Step E: App Store submission</name>
  <what-built>App is on TestFlight and verified working. This step completes the App Store listing and submits for Apple review.</what-built>
  <how-to-verify>
    In App Store Connect (appstoreconnect.apple.com > My Apps > Legacy):

    1. **Screenshots**:
       - Capture screenshots on an iPhone 6.9" device (or use Simulator with iPhone 16 Pro Max)
       - Required sizes: 6.9" display (1320 x 2868 px) — at least 3 screenshots
       - Recommended screens: landing page, dashboard with habits, habit detail with AI plan

    2. **App metadata**:
       - Description: Write a clear, honest description of Legacy (e.g. "Legacy helps Muslims carry their Ramadan spiritual habits into everyday life. Set your habits, receive a personalized 28-day plan, and stay accountable with your circle.")
       - Keywords: ramadan, habits, islam, quran, prayer, fasting, shawwal, streak, accountability
       - Age Rating: 4+ (no objectionable content)
       - Privacy Policy URL: https://legacy-bice.vercel.app/privacy (the /privacy page created in plan 06-02)
       - Support URL: https://legacy-bice.vercel.app

    3. **Review notes** (App Review Information section):
       - Add test credentials so Apple reviewers can log in without creating an account:
         - Sign-in method: Google OAuth
         - Note: "This app uses Google OAuth — reviewers can use the provided test account: [add a real Google account email/password here] or create a new Google account to sign in."
       - Add any notes about the app: "This is a habit tracker for post-Ramadan spiritual practice. Core features: habit tracking, AI-generated 28-day plans, accountability circles."

    4. **Submit for review**:
       - Select the TestFlight build to submit
       - Click "Submit to App Review"
       - Monitor review status (typically 1-3 business days)
       - If rejected: read the rejection reason carefully, fix the issue, resubmit

    Verify:
    - App Store listing shows all metadata complete (no yellow warnings in App Store Connect)
    - Submission status changes to "Waiting for Review"
    - After approval: status changes to "Ready for Sale" and app appears on App Store

    Type "live" when the app is approved and live on the App Store.
