Create a new migration file. Wrap everything in BEGIN/COMMIT. Structure:

**Section 1 — Add habit_log_id to halaqa_reactions**
```sql
ALTER TABLE halaqa_reactions
  ADD COLUMN IF NOT EXISTS habit_log_id UUID REFERENCES habit_logs(id) ON DELETE SET NULL;
```
Nullable so old reactions (without log context) remain valid. ON DELETE SET NULL prevents orphan issues if a log is ever deleted.

**Section 2 — get_circle_feed(p_halaqa_id UUID) SECURITY DEFINER RPC**

Function signature: `CREATE OR REPLACE FUNCTION get_circle_feed(p_halaqa_id UUID)`
Returns: `TABLE(type TEXT, user_id UUID, display_name TEXT, habits JSONB, streak INTEGER, created_at TIMESTAMPTZ, habit_log_id UUID, reactions JSONB)`

Security:
- `SECURITY DEFINER` — executes as function owner (bypasses RLS on habit_logs)
- First statement inside: verify caller is a member — `IF NOT EXISTS (SELECT 1 FROM halaqa_members WHERE halaqa_id = p_halaqa_id AND user_id = auth.uid()) THEN RAISE EXCEPTION 'Not a member'; END IF;`

Window: `p_window TIMESTAMPTZ := now() - INTERVAL '48 hours'` (local variable)

**Three CTEs then UNION ALL:**

CTE 1 — `log_rows`: habit check-ins in window
```sql
SELECT
  'log'::TEXT AS type,
  hl.user_id,
  p.preferred_name AS display_name,
  jsonb_agg(jsonb_build_object('name', h.name, 'icon', h.icon) ORDER BY h.name) AS habits,
  COALESCE(s.current_streak, 0) AS streak,
  MAX(hl.created_at) AS created_at,
  MIN(hl.id) AS habit_log_id   -- representative log id for reactions
FROM habit_logs hl
JOIN profiles p ON p.id = hl.user_id
JOIN habits h ON h.id = hl.habit_id
LEFT JOIN streaks s ON s.user_id = hl.user_id
WHERE hl.created_at >= p_window
  AND hl.completed = true
  AND hl.habit_id IN (
    SELECT DISTINCT hl2.habit_id FROM habit_logs hl2
    JOIN halaqa_members hm2 ON hm2.user_id = hl2.user_id
    WHERE hm2.halaqa_id = p_halaqa_id
  )
  AND hl.user_id IN (
    SELECT user_id FROM halaqa_members WHERE halaqa_id = p_halaqa_id
  )
GROUP BY hl.user_id, hl.date, p.preferred_name, s.current_streak
```

CTE 2 — `milestone_rows`: members whose streak is a milestone AND last_completed_date = today
```sql
SELECT
  'milestone'::TEXT AS type,
  hm.user_id,
  p.preferred_name AS display_name,
  '[]'::JSONB AS habits,
  s.current_streak AS streak,
  (CURRENT_DATE || 'T00:00:00Z')::TIMESTAMPTZ AS created_at,
  NULL::UUID AS habit_log_id
FROM halaqa_members hm
JOIN profiles p ON p.id = hm.user_id
JOIN streaks s ON s.user_id = hm.user_id
WHERE hm.halaqa_id = p_halaqa_id
  AND s.current_streak IN (7, 14, 21, 28, 30)
  AND s.last_completed_date = CURRENT_DATE
```

CTE 3 — `joined_rows`: members who joined within the 48hr window
```sql
SELECT
  'joined'::TEXT AS type,
  hm.user_id,
  p.preferred_name AS display_name,
  '[]'::JSONB AS habits,
  0 AS streak,
  hm.joined_at AS created_at,
  NULL::UUID AS habit_log_id
FROM halaqa_members hm
JOIN profiles p ON p.id = hm.user_id
WHERE hm.halaqa_id = p_halaqa_id
  AND hm.joined_at >= p_window
```

CTE 4 — `reaction_counts`: aggregate reactions per habit_log_id
```sql
SELECT
  habit_log_id,
  jsonb_object_agg(emoji, cnt) AS reaction_counts
FROM (
  SELECT habit_log_id, emoji, COUNT(*) AS cnt
  FROM halaqa_reactions
  WHERE halaqa_id = p_halaqa_id
    AND habit_log_id IS NOT NULL
    AND created_at >= p_window
  GROUP BY habit_log_id, emoji
) sub
GROUP BY habit_log_id
```

Final SELECT: UNION ALL the three row types, LEFT JOIN reaction_counts on habit_log_id, ORDER BY created_at DESC.

```sql
SELECT
  r.type, r.user_id, r.display_name, r.habits, r.streak, r.created_at, r.habit_log_id,
  COALESCE(rc.reaction_counts, '{}'::JSONB) AS reactions
FROM (
  SELECT * FROM log_rows
  UNION ALL SELECT * FROM milestone_rows
  UNION ALL SELECT * FROM joined_rows
) r
LEFT JOIN reaction_counts rc ON rc.habit_log_id = r.habit_log_id
ORDER BY r.created_at DESC;
```

Function attributes: `LANGUAGE plpgsql SECURITY DEFINER SET search_path = public`
